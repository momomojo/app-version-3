    JSON Format Guide

    Basic JSON Format

    Each exam file should be a JSON array of question objects. Each question object requires text, options, and correctAnswer fields. 

    [
  {
    "text": "Question text goes here...",
    "options": {
      "A": "First option text",
      "B": "Second option text",
      "C": "Third option text",
      "D": "Fourth option text", // More options can be added
    },
    "correctAnswer": "C", // The letter matching the key in options
    "explanation": "Optional explanation text for the correct answer...",
    "image": null // Optional path to an image file
  },
  // ... more question objects
]

    Required Fields:

    • text (String): The question stem.
    • options (Object): Keys are option letters (A, B, C...), values are the option texts (String).
    • correctAnswer (String): The letter key corresponding to the correct option in the options object.

    Optional Fields:

    • explanation (String): A simple text explanation for the answer. (Note: Complex explanation objects are not supported).
    • image (String or null): Path to an image file, relative to the location of the JSON file (e.g., "img/diagram.png"), or null if no image.

    Advanced Features

    You can enhance your questions with these additional features:

    Adding Images

    Include the image field with a path relative to the JSON file. Create a subdirectory (e.g., img) next to your JSON file to store the images. 

    {
  "text": "What pathology is shown in this brain MRI?",
  "image": "img/brain_mri.jpg", // Image path relative to JSON file
  "options": {
    "A": "Meningioma",
    "B": "Glioblastoma",
    "C": "Multiple Sclerosis",
    "D": "Stroke"
  },
  "correctAnswer": "B",
  "explanation": "The image shows characteristic features of glioblastoma..."
}

    Alternative Options Format (Array)

    Instead of an object, options can be an array of strings. If you use an array, correctAnswer must be the 0-based numeric index of the correct option in the array. 

    {
  "text": "Which antibiotic is appropriate for this case?",
  "options": [
    "Amoxicillin",    // Index 0
    "Ciprofloxacin",  // Index 1
    "Vancomycin",     // Index 2
    "Metronidazole",  // Index 3
    "Azithromycin"    // Index 4
  ],
  "correctAnswer": 2, // Corresponds to "Vancomycin"
  "explanation": "Vancomycin is appropriate because..."
}

    Complete Example

    [
  {
    "text": "A 45-year-old man presents with fever... Which of the following is the most likely diagnosis?",
    "options": {
      "A": "Herpes simplex encephalitis",
      "B": "Lyme disease",
      "C": "Neurosarcoidosis",
      "D": "Neurosyphilis", 
      "E": "Tuberculous meningitis"
    },
    "correctAnswer": "B",
    "explanation": "The clinical presentation and CSF findings are consistent with Lyme disease..."
  },
  {
    "text": "An 18-year-old female presents with a first seizure. MRI shows the lesion... What is the diagnosis?",
    "image": "img/cavernoma.jpg", // Image path relative to this JSON file
    "options": [
      "Astrocytoma",             // Index 0
      "Cavernous hemangioma",    // Index 1
      "Multiple sclerosis",      // Index 2
      "Arteriovenous malformation" // Index 3
    ],
    "correctAnswer": 1, // Index 1 is "Cavernous hemangioma"
    "explanation": "The image shows a cavernous hemangioma (cavernoma)..."
  }
]

    File Structure Tips

    • Save your file with a .json extension.
    • Ensure the JSON is valid (use an online validator if unsure).
    • Place images in a subdirectory (e.g., img/) located in the same directory as your JSON file. Reference them using relative paths (e.g., "img/your_image.png").
    • You can select multiple JSON files or a directory containing multiple JSON files in the welcome screen to combine questions.
